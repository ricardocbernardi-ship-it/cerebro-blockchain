1. Resumo Executivo

CEREBRO é uma blockchain proof-of-work de nova geração desenvolvida do zero em Node.js. Combina o modelo de segurança do Bitcoin com funcionalidades avançadas como smart contracts JavaScript, NFTs nativos (CBRC-721), tokens fungíveis (CBRC-20), um DEX AMM nativo e staking integrado — tudo em uma única camada.

Missão: Criar a blockchain mais acessível do mundo — onde qualquer desenvolvedor JavaScript pode criar tokens, NFTs e contratos inteligentes em minutos, sem aprender Solidity ou Rust.

O token nativo CBR alimenta toda a economia: paga mineradores, acumula rendimentos no staking, e serve como par base em todos os pools de liquidez do DEX.

2. O Problema

As blockchains existentes apresentam barreiras significativas:

2.1 Complexidade Técnica

Ethereum exige aprender Solidity. Solana exige Rust e BPF. Polkadot exige Substrate/Rust. A maioria dos desenvolvedores do mundo usa JavaScript — mas nenhuma blockchain principal permite escrever contratos diretamente em JS.

2.2 Taxas de Gas

Ethereum pode cobrar $50-$200 por transação em momentos de pico. Isso exclui casos de uso de baixo valor e cria experiência de usuário ruim para aplicações do mundo real.

2.3 Infraestrutura Fragmentada

DEX, NFTs e staking em Ethereum requerem contratos separados, múltiplas auditorias e integrações complexas. CEREBRO oferece tudo nativo, sem contratos adicionais.

4. Arquitetura Técnica

CEREBRO é composta por 5 camadas:

Camada 1 — Protocolo Base

Block.js + Blockchain.js: gerenciam a cadeia, mineração PoW, validação e persistência.

Camada 2 — Transações

Transaction.js: suporta 11 tipos de transação: TRANSFER, TOKEN_DEPLOY, TOKEN_TRANSFER, NFT_MINT, NFT_TRANSFER, CONTRACT_DEPLOY, CONTRACT_CALL, STAKE, UNSTAKE, SWAP, ADD_LIQUIDITY.

Camada 3 — Estado

Estado computado on-the-fly a partir da cadeia. Mapas internos: tokens, nfts, contracts, liquidity, stakes.

Camada 4 — API REST + WebSocket

API.js: 30+ endpoints REST. WebSocket na mesma porta para atualizações em tempo real (blocos, txs, NFTs, eventos de contrato).

Camada 5 — SDK e Ferramentas

cerebro.js: SDK JavaScript com todos os métodos. Explorer e Wallet web inclusos.

5. Mecanismo de Consenso

CEREBRO usa Proof of Work (PoW) com hash SHA-256, idêntico ao Bitcoin, mas com parâmetros otimizados para finalidade rápida:

ParâmetroBitcoinCEREBRO
AlgoritmoSHA-256SHA-256
DificuldadeAjuste dinâmico (~78 zeros)2 zeros (ajustável)
Tempo de bloco alvo10 minutosSub-segundo (instant mode)
Halving210.000 blocos210.000 blocos
Recompensa inicial50 BTC → 3.125 BTC50 CBR → diminui
Suprimento máximo21.000.000 BTC21.000.000 CBR

Modo Instant

Com instantMode = true, cada transação submete automaticamente um bloco. Isso dá finalidade sub-segundo sem sacrificar a estrutura PoW — ideal para dApps interativos.

6. Padrões de Token

CBRC-20 — Tokens Fungíveis

Equivalente ao ERC-20 do Ethereum. Parâmetros: name, symbol, supply, decimals. Deploy via transação TOKEN_DEPLOY. Transfer via TOKEN_TRANSFER.

# Deploy CBRC-20 POST /api/token/deploy { "name": "MeuToken", "symbol": "MTK", "supply": 1000000, "decimals": 18, "privateKey": "..." }

CBRC-721 — NFTs

Equivalente ao ERC-721 do Ethereum. Cada NFT tem tokenId único (UUID), nome, descrição, imageUrl, coleção e dono. Transferência on-chain. Marketplace nativo em /marketplace.html.

# Mint CBRC-721 POST /api/nft/mint { "name": "Arte Genesis #1", "description": "Primeiro NFT da CEREBRO", "imageUrl": "https://...", "collection": "Genesis", "privateKey": "..." }

7. Smart Contracts

CEREBRO é a única blockchain que executa smart contracts em JavaScript puro via sandbox Node.js (vm.runInNewContext). Sem nova linguagem para aprender.

Como funciona

O código do contrato é armazenado on-chain na transação CONTRACT_DEPLOY. Chamadas via CONTRACT_CALL executam a função especificada em um contexto isolado com acesso à função emit() para eventos.

// Exemplo: contrato de votação function votar(candidato) { const votos = state.votos || {}; votos[candidato] = (votos[candidato] || 0) + 1; state.votos = votos; emit("Voto", { candidato, total: votos[candidato] }); return votos; } function resultado() { return state.votos || {}; }

Segurança

O sandbox VM do Node.js isola a execução. O contrato não tem acesso ao filesystem, rede ou módulos do Node — apenas às funções expostas pelo contexto CEREBRO (emit, state).

8. DEX Nativo (AMM)

CEREBRO tem um AMM (Automated Market Maker) nativo no protocolo, sem precisar de contrato externo. Usa a fórmula do Uniswap v1: x * y = k.

Fórmula

// Constant product AMM x * y = k output = (y * input * 997) / (x * 1000 + input * 997) // 0.3% de taxa para provedores de liquidez

10. Economia do Token CBR

Distribuição

Destinação%QuantidadeFonte
Mineradores100%21.000.000 CBREmissão PoW (halving)
Fundador (faucet)Bootstrap~1.000 CBRFaucet inicial

Não há pré-venda, não há tokens de fundadores reservados. 100% dos CBR são emitidos pelos mineradores via PoW — igual ao Bitcoin.

Inflação

A recompensa por bloco começa em 50 CBR e é reduzida à metade a cada 210.000 blocos (Bitcoin-like). O suprimento máximo é de 21.000.000 CBR — nunca será ultrapassado.
